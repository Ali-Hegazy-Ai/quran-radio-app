# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | ✅ Yes             |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email the maintainer or use GitHub's [private vulnerability reporting](https://github.com/Ali-Hegazy-Ai/quran-radio-app/security/advisories/new)
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

## Scope

This project is a static web application that streams audio from a third-party source. Security concerns may include:

- **XSS** — Cross-site scripting via user inputs
- **CSP bypasses** — Content Security Policy misconfigurations
- **Dependency vulnerabilities** — Outdated npm packages
- **Data leaks** — Unintended exposure of environment variables

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Assessment**: Within 1 week
- **Fix**: As soon as possible, depending on severity
